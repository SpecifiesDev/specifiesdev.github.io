Features

What's new in this version? Well considering this is the first version, there's quite a lot I have implemented.
Feature Usage Description
Crop Configuration Users can add pretty much any crop in the game they want in this version. Crops have customizable growth times, experience values, random drop rates, sell prices, and more. The config.yml is robust is easy to understand. At the end of this page will have a youtube link pointing to a full setup guide.
Scoreboard In-Game If you toggle the scoreboard setting in the config.yml spigot will show a robust scoreboard with information on what season it is, time until the next season, level progress, and the amount of starcoins a player has.
Advanced Crop Deletion Backend The plugin actively works to try and prevent crops from being destroyed while at the same time ensuring a crop is removed from the stack if it is somehow destroyed to prevent lag.
Levels Config & /levels Command & Configuration You can set custom level rates in the config that will exponentially scale using my formula. Players can then use /levels in game and see the progress needed up to level 10.
Levels Backend Whenever a player harvests a crop they receive a configured amount of experience. In later versions levels are going to unlock starred drops (more valuable crops), custom items, and machines.
Holograms Feature The plugin is dependent on HolographicDisplays. It will NOT work without this plugin. I'm going to add hologram toggling in future versions, but for now they are a requirement.
/sellall Command Whenever a player gets ready to sell their produce, they can run this command. They place anything they want to sell in the inventory. If the item can be sold, they are given the StarCoins, if not it will be returned to their inventory.
StarCoins Currency StarCoins are the currency of the game. They are used to buy items from the shop, and can be used to buy machines, and other things in the future.
Crop Obtaining Feature The only way for players to actually get crop seeds is to break grass blocks. You set the chance to receive the seed in the configuration. I did it this way to sort of give that "rarity" feeling like stardew does with certain crops.
Seasons Feature Seasons are a big part of the plugin. They determine when crops can and will not grow. If the player plants the crop too late it will welter. In future version I'm going to add a gui where players can pull up crop information like growth time and seasons.

Commands

An in-depth look at the commands added.
Command Description Permissions
/giveseed /giveseed [player] [seed_name] Used to give a seed to a target player. The seed id's are defined by the "name" config value. So for example, sweet_potato's name is "&6Sweet Potato". In order to give myself the seed I'd execute /giveseed csdad Sweet_Potato betterfarming.giveseed - Blanket Permission allowing anyone with it to run the command.
/levels Opens a GUI display the player's progress up to level 10. Also outlines xp required for each level. None. This isn't an operator command.
/sellall Opens a GUI where a player can put their produce into sell. None. This isn't an operator command.
