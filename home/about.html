Skills

Over the years I have messed with a variety of languages and frameworks. The notable languages that I have dabbled with are labeled below with an organized bar graph to outline how skilled I am at them. Note that while verbose languages like SQL are not listed, I still use them as most of my development work is in backend systems.

For libraries / frameworks, I will start with Java. Most of my Java development was done in connection to Spigot, an open source API designed to allow developers to create add-ons to the close sourced game Minecraft. Through this Development I've used libraries like HikariCP to pool SQL connections, JSON libraries for web integration, and used my knowledge of networking to create a web hook like system to allow data to emit across specified channels. For front-end web development I've used popular responsive design frameworks like bootstrap to create web pages. While I've come a long way in terms of front-end design, I could come much further. I've used React in adjacency to bootstrap to create responsive web pages in an efficient work flow. I've used Electron to integrate web pages into a desktop application. I've also used React Native to design mobile applications. While a lot of my initial effort in web development was in front-end systems, I quickly moved to backend development using node.js. The most notable node framework I've used, and continue to use, is express.js. Express has taught me integral design techniques, especially when it comes to developing APIs. Through express I build my backend applications with one thing in mind: portability. The server serves data that any modern language can parse, and manipulate, thus making the platforms endless. Express has also taught me basic cyber security practices such as never raw storing sensitive data, establishing a line of encryption, and securing your source. I've also used mongoose to integrate with an Atlas MongoDB server, sql libraries, and discord.js to create discord bots for various tasks.

In terms of python, I've primarily used it for scripting and machine learning. Most of my machine learning was just me dabbling in the area and learning core matematical concepts like linear regression and stochastic gradient descent to reduce cost functions. The most complex ML project I've ever created was one that web filtered and learned bypasses, but it was just a learning project. I've never professionally developed anything with ML. I used C# in a few game development projects, but really only rate myself at the level I do because the syntax is very close to Java. I'm trying to get into C++ as I feel it's important to learn low level languages, but still prefer using high level languages for professional development environments.

In terms of real world work, the only relevant position I've held was a student worker position in my school system's IT department. While a lot of my duties involved physical tasks, I did get heavily involved during the networking side of everything towards the end of the job.

Description

A few years ago Mineplex had a public API but removed it for unknown reasons. I primarily developed this API and deployed it for a discord bot to be able to quickly display stats. While the discord bot was discontinued due to the fact that the one already out there improved I, alongside other devs, use the API in order to parse player data. For example, a bot I created has a command that sets the level of a joined user by taking the given name and querying the API for xp, and level.
