For libraries / frameworks, I will start with Java. Most of my Java development was done in connection to Spigot, an open source API designed to allow developers to create add-ons to the close sourced game Minecraft. Through this Development I've used libraries like HikariCP to pool SQL connections, JSON libraries for web integration, and used my knowledge of networking to create a web hook like system to allow data to emit across specified channels. For front-end web development I've used popular responsive design frameworks like bootstrap to create web pages. While I've come a long way in terms of front-end design, I could come much further. I've also used React Native to design mobile applications. While a lot of my initial effort in web development was in front-end systems, I quickly moved to backend development using node.js. The most notable node framework I've used, and continue to use, is express.js. Express has taught me integral design techniques, especially when it comes to developing APIs. Through express I build my backend applications with one thing in mind: portability. The server serves data that any modern language can parse, and manipulate, thus making the platforms endless. Express has also taught me basic cyber security practices such as never raw storing sensitive data, establishing a line of encryption, and securing your source. I've also used mongoose to integrate with an Atlas MongoDB server, sql libraries, and discord.js to create discord bots for various tasks.
